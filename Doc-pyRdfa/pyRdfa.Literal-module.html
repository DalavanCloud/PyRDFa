generate_literal(node, graph, subject, state)

source code 

Generate the literal, taking into account datatype, etc. Note: this method is called only if the @property is indeed present, no need to check.

This method is an encoding of the algorithm documented task force's wiki page.

The method returns a value whether the literal is a 'normal' literal (regardless of its datatype) or an XML Literal. The return value is True or False, respectively. This value is used to control whether the parser should stop recursion. This also means that that if the literal is generated from @content, the return value is False, regardless of the possible @datatype value.

Parameters:
  • node - DOM element node
  • graph - the (RDF) graph to add the properies to
  • subject - the RDFLib URIRef serving as a subject for the generated triples
  • state (state.ExecutionContext) - the current state to be used for the CURIE-s
Returns: Boolean
whether the literal is a 'normal' or an XML Literal (return value is True or False, respectively). Note that if the literal is generated from @content, the return value is False, regardless of the possible @datatype value.
